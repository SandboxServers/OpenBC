# Data Format Guide

OpenBC uses two data formats with distinct roles. The dividing line is simple: **who writes it?**

## The Split

| Format | Purpose | Who Writes | Examples |
|--------|---------|------------|---------|
| **JSON** | Machine-generated game data | Scraping tools, build pipeline | `vanilla-1.1.json`, ship stats, weapon properties, hash manifests |
| **TOML** | Human-authored configuration | Server operators, modders | `server.toml`, game modes, missions, module config, event bindings |

## JSON: Machine-Generated Data

JSON is used for data that is **extracted, computed, or generated by tools**. Nobody hand-edits these files. They are produced by scrapers, build scripts, or the hash manifest tool.

### Ship/Weapon Data

The vanilla game data lives in `data/vanilla-1.1/`. This is machine-generated by `tools/scrape_bc.py` from the game's readable script files. It contains:

- Ship class definitions (hull HP, subsystem loadouts, weapons, species IDs)
- Projectile definitions (damage, speed, blast radius)
- Map definitions (names, system indices)

Example (excerpt from ship data):

```json
{
  "species_id": 0,
  "name": "Galaxy",
  "hull_hp": 4000.0,
  "subsystems": [
    { "name": "Hull", "type": "hull", "max_hp": 4000.0 },
    { "name": "Forward Shield", "type": "shield", "max_hp": 6000.0 }
  ],
  "weapons": [
    { "name": "Phaser Array 1", "type": "phaser", "damage": 100.0 }
  ]
}
```

### Hash Manifests

Generated by `openbc-hash` from a BC installation. Contains file paths and their hash values for checksum validation.

```json
{
  "version": "vanilla-1.1",
  "files": [
    { "path": "scripts/App.pyc", "hash": "a1b2c3d4" }
  ]
}
```

### When to Use JSON

- Data produced by a scraping/extraction tool
- Data that has hundreds of entries (16 ships, 15 projectiles, thousands of checksums)
- Data that changes only when the source game files change
- Data that no human will ever hand-edit

## TOML: Human-Authored Configuration

TOML is used for files that **humans write and maintain**. Server config, game mode definitions, mission files, module manifests, event bindings.

### Server Configuration

`server.toml` is the main server config file:

```toml
[server]
port = 22101
max_players = 6
name = "My OpenBC Server"
log_level = "info"

[game]
map = "Multiplayer.Episode.Mission1.Mission1"
system = 1
time_limit = -1
frag_limit = -1
collision_damage = true
friendly_fire = false

[data]
registry = "data/vanilla-1.1/"

[[modules]]
name = "combat"
dll = "modules/combat.dll"
[modules.config]
collision_damage = true
friendly_fire = false

[[modules]]
name = "scoring"
dll = "modules/scoring.dll"
```

### Game Mode Definitions

```toml
# modes/deathmatch.toml
[mode]
name = "Deathmatch"
description = "Free-for-all, last ship standing"
teams = false
default_frag_limit = 10
default_time_limit = 15

[[handlers]]
event = "ship_killed"
module = "scoring"
handler = "on_ffa_kill"
priority = 50
```

### Mission Definitions

```toml
# missions/asteroid-field.toml
[mission]
name = "Asteroid Field FFA"
mode = "deathmatch"
system = 1

[mission.overrides]
frag_limit = 15
time_limit_minutes = 10
collision_damage = true
```

### When to Use TOML

- Server operators write it (server config)
- Modders write it (game modes, missions, module manifests)
- Small number of entries, human-readable structure
- Configuration that changes per-server or per-mod
- Event handler bindings, priority assignments

## Lua: Runtime Scripting

Lua scripts provide runtime behavior for mods. They are loaded by the Lua runtime module:

```toml
[[modules]]
name = "bonus_scoring"
lua = "mods/my-mod/scoring.lua"
```

Lua scripts are **not** a data format -- they are executable code. See [Lua Scripting Guide](../modding/lua-scripting-guide.md).

## Summary Decision Table

| Question | Format |
|----------|--------|
| Is it generated by a tool? | JSON |
| Does a human write it? | TOML |
| Is it runtime behavior (code)? | Lua |
| Ship stats, weapon data? | JSON |
| Server settings? | TOML |
| Game mode definitions? | TOML |
| Hash manifests? | JSON |
| Module config? | TOML |
| Mod scoring rules? | Lua (with TOML config) |
